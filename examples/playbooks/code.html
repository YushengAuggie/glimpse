Make the password check constant-time

The old comparison short-circuits on the first differing byte, leaking timing. Put the path and reason right next to the code.

The change

auth/service.py · lines 30–34diff · python
-    if pw_hash != user.pw_hash:
+    if not hmac.compare_digest(pw_hash, user.pw_hash):
         raise Unauthorized()
     return self.sessions.mint(user.id)
Why: hmac.compare_digest compares in constant time, so an attacker can't learn the hash prefix from response latency.

Before / after in context

beforepython
def verify(self, email, pw):
    user = self.users.find(email)
    pw_hash = hash_pw(pw, user.salt)
    if pw_hash != user.pw_hash:   # leaks timing
        raise Unauthorized()
    return self.sessions.mint(user.id)
afterpython
def verify(self, email, pw):
    user = self.users.find(email)
    pw_hash = hash_pw(pw, user.salt)
    if not hmac.compare_digest(pw_hash, user.pw_hash):
        raise Unauthorized()
    return self.sessions.mint(user.id)
