Why Glimpse

Three things a terminal can't do.

A terminal can't show your agent what it built, let it drive a real app, or let you point at the screen and reply. A real Chrome can.

1Real Chrome over CDP

It doesn't just render. It drives your live app.

Glimpse renders through a real Chrome, so the same connection also opens your running app. Your agent inspects the live page: state, console, network and all, which a static-artifact tool never sees.

  • Inspect, read-only: read a page's text + console + uncaught errors, capture a shot, or take an a11y-tree snapshot.
  • Interact: click, scroll, and wait for elements. These are the only intentionally state-changing verbs, never a side effect of reading.
  • A dedicated profile: the CDP Chrome doesn't see your everyday logins or tabs, only what you load into it.
live-app review
# open the app under review in the CDP Chrome
glimpse open http://localhost:3000

# read the rendered page + console + errors
glimpse read
→ {title, url, text, console, errors}

# drive it, then verify what the user sees
glimpse click "button.checkout"
glimpse wait --text "Order placed"
glimpse shot /tmp/after.png

How the round-trip works

When you ask, the question is captured with a text-quote anchor1: the exact words plus a little surrounding context, so the highlight can be re-found even after the document re-renders.

YouWhy anchor to the selection, not store DOM offsets?
AgentDOM offsets break the moment the page re-renders. A quote + context survives that, so your highlight stays put.
2Two-way · durable threads

Highlight anything. Ask in place. It sticks.

Feedback isn't a chat log that scrolls away. Select any passage and ask: the answer threads in the margin, anchored to your highlight and saved per document, so it survives refreshes, restarts, and fresh sessions.

  • glimpse poll is one blocking call an agent parks on. It waits for feedback, prints a token-efficient record, and returns.
  • glimpse ask flips it around: the agent publishes an interactive form and blocks until you approve, pick, or note.
  • No inbound endpoint. The bridge pulls questions over the CDP channel that's already open. Nothing new listens on the network.
3Local · private · serverless

Private by default. Nothing leaves your machine.

Everything runs on your machine: a static file server and a Chrome window, both on loopback. No account, no hosted server, no telemetry. An optional macOS menu-bar daemon can keep answering even when you're not at the keyboard.

  • Sandboxed artifacts: each renders in an allow-scripts-only iframe (opaque origin), so artifact JS can't reach the shell or a sibling.
  • Egress is loud, never silent: feedback is treated as untrusted data and secret-scrubbed; share prints a notice before anything leaves your machine.
  • Always-on, optional: glimpse daemon auto-answers via an Anthropic-compatible proxy: Q&A only, no tools, writes nothing but the answer.

Local & private by design

No walls of terminal text, and no data you didn't send. The one part that can call out, the optional daemon, is explicit about exactly what it sends.

Don't take our word for it: the data flow, alternatives, and full threat model are in docs/DESIGN.md and SECURITY.md.

Quickstart

Running in about a minute.

One script, and the whole runtime is just Node 22+ and Google Chrome. glimpse doctor tells you exactly what's missing. macOS and Linux are first-class; on Windows use WSL or Git Bash.

1

Clone & install

The installer runs a preflight, then drops the CLI into ~/.local/bin, the canvas into ~/.glimpse, and agent skills into ~/.claude/skills. Re-running is safe.

2

Open the canvas

glimpse open starts the loopback static server, launches a dedicated Chrome, and opens the dashboard.

3

Publish & talk back

Publish an example, then select a sentence and ask. Run glimpse poll to receive it and glimpse reply to answer.

zsh — quickstart
# 1 · clone & install (CLI → ~/.local/bin, canvas → ~/.glimpse)
git clone https://github.com/YushengAuggie/glimpse.git
cd glimpse && ./install.sh

# 2 · confirm deps, then serve + launch Chrome + open the canvas
glimpse doctor
glimpse open

# 3 · publish an example artifact — it opens automatically
glimpse publish guide "How to use Glimpse" ~/.glimpse/examples/glimpse-guide.html

# optional: register the chrome-devtools MCP server for MCP-capable agents
./install.sh --mcp claude

CLI reference

Every verb, grouped.

Or just talk: Glimpse ships canvas, chrome-cdp, and explain skills for Claude Code and compatible agents, so you never type the plumbing.

CommandWhat it does
Publish
glimpse open [url|#slug]Serve + launch Chrome + navigate to the canvas.
glimpse publish <slug> <title> [file]Publish an HTML artifact (stdin if no file); re-slug updates in place.
glimpse explain <slug> <title> [spec]Publish an interactive code explainer (architecture · data flow · call stack).
glimpse ask <slug> <title> [file] --formPublish interactive, block for a structured response (JSON).
Converse
glimpse poll [--json] [--timeout N]Block until there's human feedback, print it, return. One call an agent parks on.
glimpse reply <slug> "answer" --to <id>Answer a highlighted question, pinned to its anchor.
glimpse thread[s] [--json|--clear]Print one conversation thread, or list all of them.
glimpse daemonAlways-on: bridge + auto-answer via the API proxy.
Drive Chrome
glimpse read [url]Navigate + print {title, url, text, console, errors}.
glimpse shot <out.png> [url]Screenshot the current (or given) page.
glimpse snapshot [#slug|url]Print the page's accessibility tree as an indented outline.
glimpse click · scroll · waitInteract with a live page (the only state-changing verbs).
glimpse audit <slug>Report real-render layout issues (overflow / clipped / overlap).
Share & manage
glimpse export <slug> [--out path]Write a single self-contained offline HTML file.
glimpse share <slug> [--public]Upload a portable copy (private & password-protected by default).
glimpse list · rm · clear · pinKeep the sidebar tidy (pinned artifacts are always kept).
glimpse doctorCheck dependencies and running state; loud and scriptable.
